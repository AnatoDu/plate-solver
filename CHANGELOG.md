## [0.6.0] — 2026-07-07

Минорный релиз: ЕДИНАЯ параметрическая модель теорий (все четыре — пресеты
одного решателя, редукции точны по построению) и НЕЛИНЕЙНЫЙ контакт методом
обобщённой реакции ПОВЕРХ полной КТН (связки нет в литературе), многосвязные
области, диагностика зоны контакта, контактные сценарии (штамп, две пластины),
конвейер воспроизводимости и экспорт. Обратно совместимо: поведение
`classic`/`karman`/`ktn_linear`/`ktn_full` не изменено, все прежние ворота
зелёные (новые слои — чисто аддитивные). Работа велась feature-ветками
N1–N8 → `main` (каждая по зелёным воротам §0).

### Добавлено
- **Единая параметрическая модель теорий.** `theory.py` (`TheoryParams` —
  управляемые члены: `membrane`, `h_psi_sq`/`h_star_sq`, `compression`,
  `shear_field`; пресеты `classic`/`karman`/`ktn_linear`/`ktn_full`; морфинг
  `with_refinement_scale(α)`) и `ktn_solver.py` (`KTNSolver` — ОДИН решатель,
  настраиваемый пресетом; `karman` = `KarmanPlate`, `ktn_full` = `KTNPlate`
  машинно). Редукции точны ПО ПОСТРОЕНИЮ (тот же путь исполнения) — усиление
  теоремы T6.
- **Нелинейный контакт МОР+КТН.** `contact_nl.py` (`NonlinearContactMOR`): метод
  обобщённой реакции вокруг полного нелинейного `KTNSolver`; лицевое условие
  Синьорини `u_c = w + (h_c²−h_*²)Δw ≤ z` (кривизна масштабируется теорией:
  0 для classic/karman ⇒ контакт по срединной, физическая для ktn_full).
  Схемы: `nested` (эталон корректности) и `merged` (совмещённая, рабочая,
  быстрая — теорема T7, композиция сжатий МОР T4 и Пикара T5). Гейты R1/R3/R4.
- **Многосвязные области.** `geometry.make_plate_with_hole` (прямоугольник с
  круглым вырезом, `r_diff`); нелинейные теории на многосвязных `annulus`/
  `compose` (снято прежнее ограничение геометрии для karman/ktn_full).
- **Диагностика зоны контакта.** `diagnostics.py` (`contact_components` — число
  связных пятен через граф близости + union–find; `contact_report` — размер,
  доля площади, пик и локализация, суммарная сила `∫r dA`, топология);
  `NonlinearContactResult.n_components`; отрисовка профиля реакции и лицевых
  напряжений в зоне (ноутбук 07).
- **Контактные сценарии (§9.2).** Штамп: неплоское препятствие `z(x,y)` через
  поле `gap` (локализует зону под индентором). Две пластины:
  `contact_nl.NonlinearTwoPlateMOR` — взаимный контакт двух `KTNSolver`, общая
  реакция `f₁−r`/`f₂+r` (3-й закон Ньютона), непроникание лицевых
  `u_c1−u_c2≤z`; редукция к одностороннему контакту при жёсткой 2-й пластине.
- **Конвейер воспроизводимости (§9.1).** `scripts/reproduce_all.py` — «одна
  команда → все результаты»: реестр `результат → case → артефакт`, прогон
  `cases/ci`(+`--with-ladder`), сводка `results/reproduce/summary.md`, кэш.
- **Экспорт усилий и полей (§9.4).** `export.py` (`forces_on_grid` — моменты,
  реакция, мембранные `N`; `to_vtk` — legacy-VTK для ParaView без новых
  зависимостей).
- **Ноутбуки**: `07_theory_ladder` (решётка 2×2 одним решателем, морфинг α,
  подпись КТН в контакте, диагностика зоны, атлас), `08_stamp_two_plates`
  (штамп vs плоский зазор, взаимный контакт двух пластин + редукция).
- **docs**: THEORY §5-quater (единая модель + атлас различий теорий),
  ALGORITHMS §6 (совмещённая схема МОР+КТН + многосвязные ω-блоки), NOTES §22–24
  (теоремы T5 Пикар, T6 редукция, T7 совмещённый МОР–КТН), API (theory,
  ktn_solver, contact_nl, diagnostics, export, reproduce_all).

### Изменено
- **Эталонный отчёт** перегенерирован в НОВЫЙ `reference_v0.6.md` (числа золотой
  серии преемственны с v0.5 — новые слои аддитивны, `golden_config.py` не
  тронут; в отчёте изменилась лишь версия); SHA-256 в `tests/test_reference_hash.py`
  обновлён, `test_reference_run` переведён на v0.6. Исторические
  `reference_v0.3.md`/`v0.4.md`/`v0.5.md` заморожены и НЕ тронуты (§0).
- Схема композиции контакта по умолчанию — `merged` (`Config.contact_scheme`).
- Версия поднята до `0.6.0`.

### Задел
- Маршрутизация нелинейного контакта через диспетчер/case-файлы (`[model] theory`
  + профиль препятствия) и морфинг `refinement_scale` в схеме case-файлов —
  следующая версия (сейчас контакт/штамп/две пластины — прямым API `contact_nl`).
  Мягкий шарнир для КТН и алгебраические лицевые поправки `O(h²)` (`κ_q q`,
  `κ_r D r`) — отложены (сохранён доминирующий кривизный член). MMS для
  нелинейных/КТН/контактных операторов (§9.3) — средний блок, отложен.

## [0.5.0] — 2026-07-07

Минорный релиз: ПОЛНАЯ нелинейная теория Кармана–Тимошенко–Нагди в 2D, лицевые
величины первым классом, наведение порядка в таксономии теорий, редукционная
верификация и формализация алгоритмов. Обратно совместимо: поведение
`classic`/`karman`/`ktn_linear` не изменено (регресс-числа не сдвинуты). Работа
велась feature-ветками N0–N4 → `main` (каждая по зелёным воротам §0).

### Добавлено
- **Полная нелинейная КТН (`theory = "ktn_full"`).** `ktn_full.py` (`KTNPlate` —
  слой поверх `KarmanPlate`): во внеплоскостной шаг Пикара добавлены члены
  (A) `−h_*²Δq_n` и (B) регуляризация связи `+h_ψ²∫L Δv` (слабая форма Грина,
  защемление §3.5); несимметричная система — LU с диаг. предобусловливанием.
  Мягкий шарнир и Ньютон — задел (`NotImplementedError`).
- **Лицевые величины первым классом.** `faces.py` (`FaceParams`, `face_stresses`):
  прогиб лицевой `u_c`, напряжения граней (изгиб+обжатие+мембрана), интроспекция
  параметров толщины (`h_psi_sq`/`h_star_sq`/`h_c_sq`, канонические имена §3.2);
  `Result.thickness_params`, экспорт/отрисовка для любой теории.
- **Таксономия теорий.** `THEORIES = (classic, karman, ktn_linear, ktn_full)`;
  `ktn_linear` — прежнее `ktn` число-в-число; `ktn` — депрекация-алиас на
  `ktn_linear` (`DeprecationWarning`, удаление v1.0.0); `docs/MIGRATION.md`.
- **Неравномерные нагрузки.** Тип `gaussian` (аналитическая Δq) — проявляется
  член КТН (A): под неравномерной нагрузкой отличается и срединный прогиб.
- **Редукционная верификация** `tests/test_ktn_full.py` (кандидат в теорему T6):
  Gate R1 (КТН→Карман, машинная точность), R2 (→Кирхгоф), R4 (гашение O(h²/L²)),
  R5 (смыкание с ktn_linear); подпись КТН. Gate R3 (1D-КТН↔2D) отложен (§14,
  нужен 1D-эталон школы). Case-файлы ci + ladder.
- **`docs/ALGORITHMS.md`** (ГОСТ 19.701/Mermaid): псевдокод и блок-схемы ключевых
  алгоритмов (RFM-структура, Пикар+Андерсон, члены КТН, лицевые, МОР).
- Ноутбук `06_theory_comparison` обновлён под ktn_full; docs THEORY/NOTES(T6)/
  CASE_SCHEMA/API/CLAUDE.

### Изменено
- **Эталонный отчёт** перегенерирован в НОВЫЙ `reference_v0.5.md` (лестница
  включает нелинейные ступени Кармана и полной КТН `ktn_full_thickness_sweep`);
  SHA-256 в `tests/test_reference_hash.py` обновлён, `test_reference_run`
  переведён на v0.5. Исторические `reference_v0.3.md`/`reference_v0.4.md`
  заморожены и НЕ тронуты (§0); `golden_config.py` не трогался.
- Версия поднята до `0.5.0`.

## [0.4.0] — 2026-07-07

Минорный релиз: геометрически-**нелинейный** решатель по теории Кармана
(Föppl–von Kármán) и выбор теории настройкой пакета. Обратно совместимо:
поведение `classic`/`ktn` НЕ изменено (регресс-числа не сдвинуты), лестница
моделей — одним ключом `[model] theory`. Развитие в ветке `dev-postfreeze`.

### Добавлено
- **Теория Кармана (`theory = "karman"`).** `src/plate_solver/membrane.py`
  (`KarmanPlate`, `KarmanResult`): система Фёппля–Кармана в перемещениях
  (мембранная связь `L(Φ, w)`), итерация Пикара по замороженным усилиям `N_k`
  с симметричной геометрической жёсткостью `∫ N_αβ ψ_{i,α} ψ_{j,β}`, шагами по
  нагрузке и ускорением Андерсона. Канонические области (круг, квадрат),
  закрепление кромки в плане `inplane_bc ∈ {immovable, movable}`.
- **Разграничение теорий.** `THEORIES = (classic, karman, ktn)`; новый ключ
  схемы `[model] inplane_bc` и параметры нелинейной итерации (`n_load_steps`,
  `karman_relax`, `karman_max_iter`, `karman_tol`, `karman_method`) в
  `[model]`/`Config`. CLI: `--theory` / `--inplane-bc`. Значение `ktn_full`
  (полная нелинейная КТН) зарезервировано → `NotImplementedError` (релиз
  v0.5.0). Честные термины теорий задокументированы (THEORY.md, CASE_SCHEMA.md).
- **Комплекс независимых эталонов.** `src/plate_solver/benchmarks.py` —
  Hencky (мембрана), Way (ряды), Тимошенко (Ритц), Levy/IGA (ряды Фурье),
  нормировки прил. B — чистыми формулами/замороженными таблицами с
  источниками; единый источник чисел для тестов и ноутбука.
- **Верификация.** `tests/test_karman.py`: Gate L (линейный предел < 0.5 %),
  Gate M (мембранный предел Hencky, < 3 % при P̄=10³), Gate K (Way/Тимошенко/
  Levy, < 2–3 %), Gate B (immovable vs movable), воспроизводимые модульные
  тесты. Быстрые `cases/ci/karman_*.toml` (авто-CI) и тяжёлые
  `cases/ladder/karman_*.toml` (Hencky, Levy; маркер `big`).
- **Ноутбук** `notebooks/06_theory_comparison.ipynb`: три кривые
  «нагрузка–прогиб», наложенные точки эталонов и асимптота Hencky, поле
  мембранного усилия, подпись КТН в лицевых величинах.
- Мат. обоснование: эскиз теоремы T5 (сходимость нелинейной итерации),
  аргумент валидации эталонами разной природы (NOTES.md §22).

### Изменено
- **Эталонный отчёт** перегенерирован в НОВЫЙ файл `reference_v0.4.md`
  (обоснование: лестница `cases/ladder` теперь включает детерминированные
  нелинейные karman-ступени — Hencky и Levy, `reference="none"`). Строка
  SHA-256 в `tests/test_reference_hash.py` обновлена, `tests/test_reference_run.py`
  переведён на v0.4. Исторический `reference_v0.3.md` заморожен и НЕ тронут
  (протокол §0); `golden_config.py` не трогался.
- Версия поднята до `0.4.0` (единый источник `plate_solver.__version__`).

## [0.3.1] — 2026-07-06

Патч по правкам акта приёмки: документация, комментарии, ноутбуки,
эргономика вывода. Пре-тег прогон: `pytest` (все маркеры, включая

лёгкая связка — 268 тестов. Поведение решателей не изменено, эталонный отчёт
reference_v0.3.md не тронут (хеш-ворота зелёные). Узкие дополнения API
ВЫВОДА (`solve(grid_n=…)`, `Result.regrid`,
`Problem.with_discretization`) включены в патч решением автора.

### Добавлено
- Сетка вывода задаётся «на месте» и уплотняется мгновенно:
  `grid_n` в шаблонах `--new` с пояснением («на точность решения не

  реакции БЕЗ повторного МОР. Ворота: тождество на родной сетке
  побайтово; контакт на 4·n₀ без новых итераций.
- Копируемые примеры фабрики эталонов в docs/API.md (кольцо со
  смешанными кромками и сила в центре; Леви со свободными кромками;
  замкнутый контакт круг+основание) — исполняются тестом

## [0.3.0] — 2026-07-06

исправления, документация, упаковка и три санкционированных дополнения:

Пре-тег прогон: `pytest` (все маркеры big+fem, включая повторный
эталонный прогон run_reference) — **310 passed** за ~17 мин
(macOS/arm64, Python 3.14); лёгкая связка `-m "not big and not fem"` —
**261 тест** за ~70–75 с (в v0.3.0).

  контактной зоне, пишутся `null` (вместо невалидного `NaN`);
  блок контакта присутствует при `contact.enabled` всегда, включая
  случай «касания нет».
- Кросс-платформенный допуск регресса профиля σy⁻ под штампом
  (чувствительность недосошедшего МОР к BLAS-платформе).
- Кросс-платформенные ворота профиля dh (канон 21.1): dh — разность
  близких величин, у кромки зоны недосошедший ci-МОР платформозависим
  вплоть до знака малых точек; гейтятся пик, глубокая зона (|dh| ≥ 0.3
  пика) и масштаб кромочного звона (hotfix по красному CI до публикации
  тега).
