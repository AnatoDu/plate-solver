# Changelog

Формат — по [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/);
версии — [SemVer](https://semver.org/lang/ru/). Числа регресса защищены
тест-воротами (`pytest`); эталонные отчёты — под хеш-контролем.

## [0.3.1] — 2026-07-06

Патч по правкам акта приёмки: документация, комментарии, ноутбуки,
эргономика вывода. Пре-тег прогон: `pytest` (все маркеры, включая
повторный эталонный прогон) — **317 passed** за ~18 мин (macOS/arm64);
лёгкая связка — 268 тестов. Поведение решателей не изменено, эталонный отчёт
reference_v0.3.md не тронут (хеш-ворота зелёные). Узкие дополнения API
ВЫВОДА (`solve(grid_n=…)`, `Result.regrid`,
`Problem.with_discretization`) включены в патч решением автора.

### Добавлено
- Сетка вывода задаётся «на месте» и уплотняется мгновенно:
  `grid_n` в шаблонах `--new` с пояснением («на точность решения не
  влияет»); CLI `plate-solve … --grid N`; программно
  `solve(problem, grid_n=…)` и `Problem.with_discretization(p, Q,
  grid_n)`; `Result.regrid(N)` — переоценка всех полей (w, моменты/σ,
  поверхности, реакция, зона) из удержанных коэффициентов и узловой
  реакции БЕЗ повторного МОР. Ворота: тождество на родной сетке
  побайтово; контакт на 4·n₀ без новых итераций.
- Копируемые примеры фабрики эталонов в docs/API.md (кольцо со
  смешанными кромками и сила в центре; Леви со свободными кромками;
  замкнутый контакт круг+основание) — исполняются тестом
  `tests/test_api_examples.py`.

### Исправлено
- Число ворот лёгкой связки в примечании к 0.3.0: 261 тест (а не 117).
- Матрица возможностей идемпотентна (FEATURES.md исключён из пула
  само-описания; вскрытая этим дыра `--help` закрыта в README).

### Изменено
- Финальная зачистка меток внутренних планов (`F#`, `P#.#`) из
  комментариев кейсов и докстрингов; стоп-словарь doc_policy расширен
  этими паттернами (ноутбуки проверяются по source-ячейкам).
- Поставка на ревью — штатный `git archive` (процессный скрипт сборки
  архива удалён из публичного репозитория; решение автора).
- Документированы grid-зависимые диагностики (топология зоны — по сетке
  вывода; в baselines.json к топологии добавлен `grid_n`).

## [0.3.0] — 2026-07-06

Доведение до самостоятельного программного продукта (feature-freeze:
исправления, документация, упаковка и три санкционированных дополнения:
фабрика аналитических эталонов, поля лицевых поверхностей, свободный край).

Регресс: **310 тест-ворот** (лёгкая связка `-m "not big and not fem"` —
~70 с; покрытие кода лёгкой связкой — 85 %, информационно); единый эталонный отчёт `results/reference/reference_v0.3.md`
(48 чисел в csv) под SHA-256-воротами; ключевые числа: круг p-свип
(ошибка < 0.1 %, модельный разрыв 26.42 %), L-форма RFM↔Marcus 2.64 %,
контакт МОР r_max = 112.58 (67/10800 узлов), КТН пик ×0.095 / узлы ×7.75,
1D-штамп ↔ Maple 2.4 % L², 24 ladder-ступени в допусках «факт × 3».

Пре-тег прогон: `pytest` (все маркеры big+fem, включая повторный
эталонный прогон run_reference) — **310 passed** за ~17 мин
(macOS/arm64, Python 3.14); лёгкая связка `-m "not big and not fem"` —
**261 тест** за ~70–75 с (в v0.3.0).

### Добавлено
- ФАБРИКА аналитических эталонов с самосертификацией
  (`analytic_auto`): осесимметрия (полином q(r), сила в центре, кромки
  clamped|soft|true_ss), Навье uniform|patch|point, Леви с
  несимметричными и СВОБОДНЫМИ кромками, 1D-полоса; замкнутый контакт
  круг+основание (кольцевая реакция Кирхгофа; предел согласован с
  функцией Грина). Эталона без проверенного сертификата не существует.
- СВОБОДНЫЙ КРАЙ: `type = "free"` в `[[bc.sides]]` (естественные условия
  полной формы), правило жёстких смещений, эталоны Леви-free и
  Морли-МКЭ, ступени rect_sfsf / rect_cfff (консоль).
- ПОЛЯ ЛИЦЕВЫХ ПОВЕРХНОСТЕЙ: w_top / w_bot / dh в fields.npz (схема 2,
  NOTES §21), `--surface mid|top|bottom`, σ-шестёрка ВТОРОЙ пластины
  пары; NOTES §21: канон — кинематика КТН (решение автора, §21.1),
  классическое 3D-восстановление — независимая диагностика (§21.2,
  кривизный блок тождественен, разности q,r-членов — постоянные
  sympy-ворота; различие в пределах O(h²)-точности теории).
- Единый эталонный регресс: `scripts/run_reference.py` (золотая серия +
  1D-штамп + структура-против-штрафа + вся лестница) → отчёт под
  хеш-воротами; детерминизм повторного прогона — тестом.
- CLI: `--check` (валидация для CI), `--report` (одностраничный md),
  `--version`, `--surface`; примеры в `--help`.
- Документация: THEORY.md, ARCHITECTURE.md (граф импортов), API.md,
  FEATURES.md (автоматическая матрица возможностей БЕЗ дыр), cases/README;
  CASE_SCHEMA добит (exit-коды, оси/поверхности, покрытие analytic).
- Планшет контакта пары пластин с панелями w₁/w₂
  (`viz.plot_pair_summary`; штатный путь фигур выбирает его сам).
- `scripts/make_review_zip.sh` — приёмо-архив одной командой
  (исходники + лёгкий снимок полей + свежий pytest-лог).
- Упаковка: sdist+wheel (`python -m build`, twine-чистые), MANIFEST,
  classifiers/urls, CITATION.cff 0.3.0, .zenodo.json, RELEASING.md,
  CI-job build; единый источник версии — `plate_solver.__version__`.
- CHANGELOG.md (этот файл); журнал разработки перенесён в приватную часть.

### Исправлено
- Тест редукции формулы (11) — полный вид по первоисточнику:
  мембранный член T/h, изгибный и b-поправка — h̊², знак b-поправки
  противоположен изгибному.
- `result.json` — строгий JSON: метрики, не определённые на пустой
  контактной зоне, пишутся `null` (вместо невалидного `NaN`);
  блок контакта присутствует при `contact.enabled` всегда, включая
  случай «касания нет».
- Кросс-платформенный допуск регресса профиля σy⁻ под штампом
  (чувствительность недосошедшего МОР к BLAS-платформе).
- Кросс-платформенные ворота профиля dh (канон 21.1): dh — разность
  близких величин, у кромки зоны недосошедший ci-МОР платформозависим
  вплоть до знака малых точек; гейтятся пик, глубокая зона (|dh| ≥ 0.3
  пика) и масштаб кромочного звона (hotfix по красному CI до публикации
  тега).

### Изменено
- Документация переведена на язык продукта (без внутренних процессных
  привязок); научная библиография сохранена: Михайловский–Тарасов (1993),
  Рвачёв, Ермоленко–Туркова (2023) — источник формулы (11),
  Ермоленко–Дуркин (2022) — источник 1D-эталона Maple.
- Исторический «золотой» отчёт ПОГЛОЩЁН единым reference: числа
  воспроизведены теми же функциями и сверены воротами преемственности
  (все 78 расчётных чисел присутствуют побайтово в напечатанной
  точности); legacy-файлы выведены из отслеживаемой части, хеш-ворота
  перенесены на reference_v0.3.md. Пакет-фасад `plate_solver.__init__`
  экспортирует публичные точки секциями (лениво).

## [0.2.0] — 2026-07-04

Слой постановки задачи: комплексом пользуются case-файлом, а не
программированием.

- Схема case-файлов TOML (docs/CASE_SCHEMA.md): geometry / bc / load /
  model / contact / discretization / verify / output; валидатор с
  диагностикой `CaseError` («ключ: получено X, ожидалось Y»).
- Реестр геометрий: circle, rectangle, L, annulus, мини-язык compose
  (union/intersect/difference, глубина ≤ 3, ≤ 7 узлов).
- Диспетчер `solve(problem) → Result`; result.json с провенансом
  (git-хеш, версии зависимостей).
- CLI: plate-solve / plate-verify / plate-ladder, свипы `--sweep p=A:B:S`.
- Верификация как свойство постановки: `[verify]` с эталонами
  analytic / mms / fem / cross_1d; реестр cases/ci = CI-ворота.
- Эталоны: кольцо (аналитика 4×4 и 1D-Ритц), точечная сила
  (регуляризованное пятно, NOTES §18), fem (scikit-fem, Аргирис/Маркус).
- Золотой отчёт заморожен хеш-воротами (SHA-256).
- 210 тест-ворот.

## [0.1.0] — 2026-07-02

Первый рабочий выпуск: изгиб и односторонний контакт упругих пластин
произвольного очертания.

- Геометрия R-функциями (система R0 Рвачёва), структуры решения ω·Φ
  (мягкий шарнир) и ω²Φ (защемление) на базисе Чебышёва.
- Изгиб: расщепление бигармоники на две задачи Пуассона (мягкий шарнир);
  прямой Ритц по (D/2)∫(Δw)² (защемление).
- Контакт: метод обобщённой реакции r ← [r + β(w − Δ)]₊ с теоремой
  сходимости; неканонические области (L-форма с входящим углом).
- Поправки уточнённой теории КТН (сдвиг + обжатие) в изгибе и контакте.
- 1D-задел: балка-полоса, штамп (Ритц и функция Грина), эталон Maple.
- Верификационная лестница 1D↔2D↔аналитика; золотой прогон одной серией
  (scripts/run_golden.py).
- 104 тест-ворот.
