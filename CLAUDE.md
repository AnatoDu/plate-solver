# plate-solver — комплекс программ (кандидатская, спец. 1.2.2, физ-мат)

## Контекст
Это защищаемый «комплекс программ» по триаде Модель–Алгоритм–Программа
(Самарский). Тема: изгиб и односторонний контакт упругих пластин
произвольного очертания по уточнённой теории типа Кармана–Тимошенко–Нагди
(КТН). **Репозиторий ПУБЛИЧНЫЙ** и пойдёт на защиту — держим его чистым,
воспроизводимым и пригодным для использования посторонним.

## Архитектура (направление зависимостей строго вниз)
- `model/`     — соотношения КТН, постановки задач (`problems.py`)
- `geometry/`  — R-функции: граница ω(x,y)=0, операции R0 (Рвачёв)
- `solver/`    — бигармоника / Пуассон на произвольной области
- `contact/`   — метод обобщённой реакции (МОР), свободная граница
- `analytic/`  — 1D-решения круглой пластины (только как верификатор)
- `tests/`     — автотесты, включая верификацию 1D↔2D
- `examples/`  — минимальные воспроизводимые примеры

## Команды
- Установка: `pip install -e ".[dev]"` (+ `.[dev,fem]` для МКЭ-верификации)
- Тесты: `pytest -m "not big and not fem"` (быстрые); полный — `pytest`
- Стиль: `ruff check .`
- Пример: `python examples/circular_plate.py`
- Золотой прогон: `python scripts/run_golden.py` (из корня; пишет в CWD)

  Это сертификат корректности всего метода; при расхождении — искать
  ошибку в методе/краевых условиях, а не в допуске.
- YOU MUST: каждый новый численный метод сопровождать тестом и кратким
  мат. обоснованием (порядок аппроксимации, предполагаемая оценка погрешности).

  `results/`, `*.npz/*.npy/*.csv` не коммитить (см. .gitignore).
- Только свободный инструментарий; зависимости минимальны и зафиксированы.
- Каждый публичный модуль/функция — с docstring; математику пояснять формулой.
- Стиль кода: ruff (line-length 100). Стиль пояснений — академический, по-русски.
- НЕ делать `git add -A` вслепую; не коммитить секреты и личные настройки.
- `private/` НИКОГДА не коммитится и не пушится (домашние материалы автора;
  каталог в .gitignore) — контроль: `git ls-files private/` обязан быть пуст.

## Связь с диссертацией
Код реализует положения на защиту № 2 (численный метод + МОР) и № 3 (комплекс
программ); верификация (положение № 4) живёт в `tests/`. Новизна — на
неканонических формах (углы, входящие углы, невыпуклость, вырезы): интеграция
R-функции + МОР + уточнённая теория для контакта со свободной границей.
