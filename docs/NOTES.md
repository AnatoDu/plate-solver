# NOTES — тонкости и подводные камни (журнал заметок)

Академические заметки к комплексу. Держим здесь всё, что легко забыть и на чём
легко ошибиться. Обновлять по мере заполнения модулей.

## 0. Соглашения о знаках (КРИТИЧНО)
- Нагрузка `q > 0` направлена «вниз», прогиб `w > 0` — тоже «вниз».
  **Контрольная проверка:** на круге при равномерной `q > 0` должно быть `w > 0`
  всюду, максимум — в центре. Если знак перевёрнут — ошибка в (P1)/(P2) или в
  делении `M/D`.
- Оператор `−Δ` (с минусом) положительно определён на `H¹₀`; именно его собирает
  `assembler.py`. Для `−Δv = f` слабая форма: `∫_Ω ∇v·∇φ dΩ = ∫_Ω f·φ dΩ`.
- Расщепление: `M = −D·Δw` ⇒ `−ΔM = D·Δ²w·(−1)?` — **выкладку держать явно**:
  `D·Δ²w = q̃`, `Δ²w = Δ(Δw)`, `Δw = −M/D`, значит `Δ²w = −ΔM/D`,
  откуда `D·(−ΔM/D) = q̃` ⇒ `−ΔM = q̃`. Это (P1). Затем `−Δw = M/D` — это (P2).
  Оба — `−Δ(·) = (правая часть) ≥ 0` ⇒ оба решения положительны при `q̃ ≥ 0`.

## 1. Расщепление бигармоники (ядро)
- Одна и та же матрица Ритца `A` для (P1) и (P2): различаются только правые части.
  ⇒ факторизуем `A` **один раз** (Холецкий, т.к. `A` симметрична и положительно
  определена), далее только обратные подстановки. Это же экономит на каждой
  итерации МОР.
- `M` — вспомогательное поле («суммарный момент»), не путать с изгибающими
  моментами `M_x, M_y`. Здесь `M = M_x + M_y` (для классики), т.е. след тензора
  моментов с точностью до множителя `(1+ν)` — для «мягкого шарнира» этого
  достаточно, т.к. на границе зануляются обе компоненты.

## 2. Обусловленность матрицы Ритца
- Полиномы высокой степени дают плохо обусловленную `A`. Чебышёв смягчает, но при
  `p > 16` (степень по оси) **следить за числом обусловленности** `cond(A)`.
- Меры при плохой обусловленности: нормировка столбцов базиса; снижение `p`;
  переход на ортогонализованный базис. Логировать `cond(A)` в сборщике.
- Базис строим на **описанном прямоугольнике (bbox)** области, отображая его в
  `[−1,1]²`, где Чебышёв естественен.

## 3. Контактная реакция `r`: источник истины
- `r` храним **в узлах квадратуры** (там, где считаем энергию и невязку). На
  сетку вывода `r` только сэмплируем для графиков. Никаких «двойных» представлений
  `r` — меньше путаницы с интерполяцией сетка↔квадратура.
- Нагрузка в (P1): `q̃ = qₙ − r`, причём `r` берётся в тех же узлах.

## 4. Маска квадратуры у входящего угла L-формы
- Тензорная квадратура Гаусса–Лежандра по bbox + маска `ω > 0`. У входящего
  (реентрантного) угла возможна локальная потеря точности интегрирования и
  «ступенька» маски.
- Контроль: сгущать квадратуру у угла; сравнивать интегральные характеристики и
  поле с scikit-fem (шаг верификации). Ожидаемая особенность решения у входящего
  угла: `r^{2/3}`-сингулярность производных — полиномами Чебышёва ловится плохо,
  это известное ограничение метода.
- **Количественно (диск, −Δv=1).** Точное решение `v=(1−r²)/4` лежит в структуре
  `ω·Φ` (Φ≡1/2), поэтому ошибка аппроксимации ~0, а остаток задаётся СТУПЕНЧАТОЙ
  маской: отн. L²-погрешность падает как `~0.98/Q` (Q=64 → 0.77%, Q=128 → 0.5%,
  Q=256 → 0.38%, Q=1024 → 0.096%). Вывод: для <0.1% нужен `Q≈1024–1280` при
  умеренном `p`. С ростом `p` ошибка падает лишь слабо (предел задаёт квадратура,
  а не базис). Это влияет на достижимую точность изгиба и контакта — учитывать
  при выборе `Q`.

## 5. Скорость и устойчивость МОР
- Итерация (перенос из `fix_base2.py`, 1D-классика, без изменений по сути):
  ```
  w ← (решатель)(q̃ = q − r)          # дёшево: A уже факторизована
  r ← max(r + β·(w − Δ), 0)           # только в зоне возможного контакта
  ```
  где `Δ` — зазор до основания/штампа, `β` — параметр релаксации.
- **Теорема 4 (условие сходимости):** `0 < β < 2/‖G‖`, где `G` — оператор
  «реакция → прогиб». Малый `β` ⇒ много итераций (в 1D были десятки тысяч), но
  благодаря факторизации это терпимо. Можно поднимать `β` и проверять устойчивость.
- Критерий останова: по норме приращения `r` (или по невязке комплементарности
  `r·(w−Δ) → 0`), а не по фиксированному числу итераций.

## 6. scikit-fem для бигармоники (верификация L-формы)
- Обычный `P1` для бигармоники **не годится** (нужна `C¹`/смешанная постановка).
  Брать **элемент Морли** или **HCT**, по официальному примеру scikit-fem
  «Kirchhoff plate». Это независимый код — именно поэтому годится как сверка.
- Сверяем: поле `w`, `w_max`, и (для контакта) зону контакта и реакцию.

## 8. Модельная погрешность «мягкого шарнира» на криволинейной границе
- «Мягкий шарнир» берёт на границе `M = −D·Δw = 0`. Истинное шарнирное опирание
  (Кирхгоф) требует зануления РАДИАЛЬНОГО момента `M_r = −D(w'' + ν w'/r) = 0`.
  На границе `Δw − M_r/(−D) = (1−ν) w'/r ≠ 0` ⇒ при `ν ≠ 1` модели расходятся.
  Расхождение ∝ `(1−ν)·кривизна`; для ПРЯМЫХ краёв (L-форма) кривизна 0 ⇒
  расщепление точно совпадает с истинным шарниром.
- **Круг, количественно.** Расщепление даёт `w = q(a²−ρ²)(3a²−ρ²)/(64D)`
  (это (4.2) с `(5+ν)/(1+ν) → 3`, т.е. (4.2) при ν=1). Отношение к Кирхгофу:
  `w_soft/w_SS = 3(1+ν)/(5+ν)`; при ν=0.3 модельная погрешность `1 − 3(1.3)/5.3 ≈
  26.4 %` и НЕ зависит от p, Q (это не числовая, а модельная ошибка).
- **Следствие для верификации (Таблица 4.1).** Сверять ЧИСЛЕННЫЙ метод с
  аналитикой РЕАЛИЗОВАННОЙ модели (`circular_plate_soft_hinge`) — там ошибка мала
  и падает с Q. Расхождение с (4.2) выносить отдельной строкой как модельную
  погрешность мягкого шарнира. (Уточнение модели — поправка КТН, §7.)

## 9. Парадокс Сапонджяна–Бабушки (входящий угол L-формы)
- Расщепление на две Пуассона (мембранная аналогия Маркуса) точно эквивалентно
  истинному шарниру Кирхгофа только для ВЫПУКЛЫХ областей. На ВХОДЯЩЕМ
  (реентрантном) угле они дают РАЗНЫЕ решения: у истинного Кирхгофа момент
  `M = −D·Δw` в реентрантном угле сингулярен/не зануляется, а расщепление
  навязывает `M = 0` на всей границе ⇒ систематическое расхождение.
- **Численно подтверждено** (L = [0,1]²\[0.5,1]², ν=0.3): RFM ↔ FEM-Marcus
  (P2, те же две Пуассона) совпадают на ~2.4 % (верификация численности), а
  RFM ↔ FEM-Kirchhoff (Морли) расходятся на ~40–50 % в L², максимум у входящего
  угла (отношение до ~2×); расхождение РАСТЁТ с измельчением сетки (FEM-Кирхгоф
  сходится к другому пределу) — это модельный, а не сеточный эффект.
- **Следствие.** Верификация на L-форме ДВУХКОЛОНОЧНАЯ (verify_fem.py): гейт —
  RFM ↔ FEM-Marcus (та же модель, мало); колонка RFM ↔ FEM-Kirchhoff выносится в
  Таблицу 4.2 как количественная мера парадокса. Корректный учёт входящего угла —
  мотивация уточнённой модели (КТН, §7) и темы «произвольное очертание».

## 10. МОР в 2D: масштаб шага β и где пик реакции
- **Масштаб β.** В 1D (``fix_base2``) задача безразмерна и β=0.01. В 2D при
  физических ``D`` прогиб мал (``w~q a⁴/D``), поэтому шаг нормируется на усиление
  оператора: ``β_eff = cfg.beta / gain``, ``gain`` = макс. прогиб от единичной
  равномерной нагрузки (оценка ‖G‖). Тогда ``cfg.beta`` — БЕЗРАЗМЕРНЫЙ коэффициент
  релаксации, теорема 4: ``0 < cfg.beta < 2``. Бигармонический оператор сильно
  НЕЛОКАЛЕН (G_active имеет большой разброс собственных значений), поэтому
  сходимость МОР МОНОТОННА, но ПОСТЕПЕННА (как 50000 итераций в 1D). В демо берём
  ``cfg.beta ≈ 1–1.5``; дефолт 0.01 — слишком медленно.
- **Где пик реакции (важно для Таблицы 4.2).** При жёстком плоском основании под
  всей пластиной контакт возникает в БУЛЬШОЙ части (там, где свободный прогиб
  максимален). Пиковая реакция — на КРОМКЕ контактного пятна (сингулярность
  давления под жёстким штампом), в зоне, прилегающей к входящему углу, но НЕ
  В САМОМ углу: реентрантный угол — точка шарнирной границы, где ``w = 0``,
  поэтому контакта там нет. Контактное пятно ИНДЕНТИРОВАНО входящим углом
  (его форма искажена), и пик тяготеет к этой области (``d ≈ 0.2`` от угла).
  Точная формулировка: пик у кромки пятна вблизи угла, не в самой вершине.

## 7. КТН (этап 6) — РЕАЛИЗОВАНО (`plate_solver/ktn.py`)
- Поправки уточнённой теории (Карман–Тимошенко–Нагди) перенесены из
  `lab/Задачи Python/2/code.py` (1D) в 2D. Базовый кирпич — то же расщепление:
  кривизна берётся ПРЯМО из (P1) как `Δw = −M/D` (без численного
  дифференцирования, в отличие от второй разности `w''` в 1D).
- Поправочные длины: `h_Ψ² = h²/(6(1−ν))` (сдвиг), `h_*² = ν h²/(8(1−ν))`
  (обжатие), `h_z² = h_Ψ²−h_*²`. Коэф. кривизны `2h_*²−h_Ψ²` при `Δw`.
- Поправки входят в КОНТАКТНОЕ условие (обновление r): смещение контактной
  поверхности `u_c = w + (2h_*²−h_Ψ²)Δw − A·q0 − B·D·r` (члены ∝q0 — «в q̃»,
  ∝r — «в обновление r»). КТН-прогиб срединной поверхности — отдельная формула
  (коэффициенты с `(3ν−1)`, `(3−ν)`), используется для поправки `w_max`.

## 11. Эффект КТН в контакте (классика ↔ КТН)
- **Сглаживание реакции.** Член ∝r (`−B·D·r`) в условии контакта действует как
  РАСПРЕДЕЛЁННАЯ ПОДАТЛИВОСТЬ ⇒ регуляризует сингулярность давления под жёстким
  штампом: классика даёт узкий высокий пик у кромки, КТН — широкое сглаженное
  пятно. Численно (L-форма, ν=0.3): при `h=0.06` пик реакции КТН/классика ≈ 0.23,
  шероховатость ≈ 0.23, узлов контакта 11 → 45. Эффект растёт с `h`: `h=0.02`
  (≈нет, режим тонкой пластины), `h=0.04` (пик ×0.71), `h=0.06` (×0.23).
- **Поправка w_max.** Уточнённая теория МЯГЧЕ (сдвиг+обжатие) ⇒ прогиб больше:
  `Δw_max` ≈ +3 % (h=0.04), +18 % (h=0.06). Знак — увеличение.
- **Режим.** Для умеренной, наглядной поправки берём `h/L ~ 0.05–0.1`; при
  `h≳0.2` КТН меняет режим (контакт может исчезнуть) — это уже не «поправка».

## 12. Жёсткое защемление: структура `ω²Φ`, прямой Ритц по бигармонике (`plate_solver/clamped.py`)
- Защемление берёт структуру `w = ω²Φ` (p=2): зануляет `w` И `∇w` на ∂Ω тождественно
  ⇒ расщепление на две Пуассона НЕ нужно (и НЕ применимо). Минимизируем
  `J = (D/2)∫(Δw)² − ∫qw`: на пространстве `ω²Φ` (где `w=0, ∇w=0` на крае)
  гауссов член `∫(w_xx w_yy − w_xy²)` зануляется тождественно, поэтому
  `(D/2)∫(Δw)²` — ТОЧНАЯ энергия, а Эйлер–Лагранж — истинная защемлённая
  бигармоника. Нужен только лапласиан `Δ(ω²T)` (не смешанные вторые производные).
- **Парадокса при защемлении НЕТ** (в отличие от шарнира). Контраст числами:
  - круг — RFM↔(4.1) ~0.09 % при ЛЮБОМ p (точное решение `w=[qa²/16D]·ω²` лежит в
    структуре уже при p=2); ошибка падает с Q (~1/Q), модельного «пола» нет
    (у мягкого шарнира 26.4 % — независимо от p, Q).
  - L-форма — RFM↔МКЭ(Аргирис, C¹) = 9.9 % L² / 5.0 % по w_max при p=10 и УБЫВАЕТ
    с p (33.7→25.6→18.0→13.2→9.9 %): это ОБЫЧНАЯ погрешность дискретизации у
    входящего угла, сходящаяся к нулю, а НЕ модельный разрыв. Парадокс шарнира
    54.86 % НЕ убывает (растёт с измельчением) — вот ключевое отличие.
- **Почему L-форма сходится медленнее круга/квадрата.** На ВЫПУКЛЫХ (круг, квадрат)
  RFM↔МКЭ ~0.06–0.17 % — метод по сути точен. Медленная сходимость на L —
  СПЕЦИФИКА входящего угла: защемлённая угловая сингулярность (показатель ≈0.5445)
  ловится полиномами Чебышёва плохо (как и `r^{2/3}` у шарнира, §4), а 4-й порядок
  оператора (вторые производные в энергии) усиливает эффект. Это разрешает p, не Q
  (Q сошлось при 120). cond(S) бигармоники РАСТЁТ как ~`(cond Пуассона)`²: p=10 →
  `~1e17`. Спасает диагональное предобуславливание (нормировка на `√S_kk`) — на
  результат не влияет (проверено), но держит решение устойчивым.
- **МКЭ для защемления = Аргирис (полный C¹, степень 5)**, доступен в scikit-fem.
  Защемление: фиксируем граничные dof `u, u_x, u_y` (⇒ ∇w=0: и нормаль, и касательная)
  и `u_n` на рёбрах; вторые производные СВОБОДНЫ (момент на крае — часть решения,
  их фиксация пересжимала бы). Проверено: круг ~0.17 %, квадрат ~0.06 %. Вариант
  «зафиксировать ВСЕ граничные dof» пересжимает (на круге 8→2 % — хуже). Морли
  (нонконформный) — запасной, сходится, но грубее (круг ~2 % при r=4).

## 13. 1D-штамп: 50000 итераций fix_base2 — это НЕДОсходимость (`plate_solver/stamp.py`)

  улучшается 7.3 %→3.3 %→2.4 % (L²). Внимание: `solve_mor_1d` по умолчанию
  `max_iter=500000` ⇒ ОСТАНАВЛИВАЕТСЯ по лимиту (не по сходимости) на ~3.3 %.
- Основной результат — СОШЕДШЕЕСЯ решение (2.44 % к Maple); 50000-шаговый снимок
  воспроизводится в тест-воротах как контроль метода (числа fix_base2 точно).

## 14. Верификационная лестница изгиба (`plate_solver/ladder.py`, `run_ladder_*.py`)
- Лестница по нарастанию сложности, RFM+Ритц БЕЗ МОР; каждая ступень — к точному
  решению/эталону (ν=0.3). Числа (квадрат, ν=0.3): синус 1.5e-6 (p10); равномерный
  шарнир w→0.00406235, M→0.0478864 (Навье 99×99 совпал с таблицей до 8 знаков);
  равномерное защемление w→0.00126532 (e1e-6), M_c→0.0229051, M_край→−0.0513338
  (e1e-4). 1D полоса — машинно (решение полиномиально).
- **1D шарнир: естественное ГУ.** Минимизация ``(D/2)∫(w'')²−∫qw`` на ``w=ω·Φ``
  (только ``w=0`` существенно) даёт ``w''=0`` на краях ЕСТЕСТВЕННО ⇒ это ИСТИННЫЙ
  шарнир (в 1D парадокса нет, см. §8). Защемление — ``w=ω²·Φ`` (``w=0`` и ``w'=0``).
- **MMS — какой ω.** Для машинной точности (изоляция сборки/квадратуры) брать ω
  ПОЛИНОМИАЛЬНУЮ: прямоугольник ``ω=(a²−x²)(b²−y²)`` (НЕ R-функция make_rectangle!),
  тогда ``w=(x²−a²)²(y²−b²)²=ω²`` лежит в структуре И квадратура Гаусса на bbox
  ТОЧНА (домен=bbox) ⇒ L²~1e-15. MMS-круг ``w=(R²−r²)²/64D=ω²·const`` — тоже в
  структуре, НО криволинейная граница ⇒ ступенчатая маска квадратуры ⇒ L²~1/Q
  (8e-3 при Q=128 → 1.2e-3 при Q=1024), НЕ 1e-6. Это и есть проверка геометрии.
- **Моменты из RFM (`bending_moments`).** ``M=−D(w_xx+νw_yy)`` через гессиан
  структуры ``ω^p·Φ``: ``w_xx=[ω^p]_xx v+2[ω^p]_x v_x+ω^p v_xx`` (для p=1 член
  ``ω^{p-2}`` гасится множителем ``p(p−1)=0`` — НЕ вычислять ``ω^{−1}``). Момент
  края — «медленная» величина (e~1e-4 при p=12), прогиб — «быстрая» (e~1e-6 при p10).
- **Обусловленность защемления зависит от ВЫПУКЛОСТИ.** Защемлённый прямоугольник
  (выпуклый) cond(S)~1.5e4 при p=10 и w_max→6 знаков; L-форма (входящий угол) —
  cond~1e17 и ~5 % (§12). Дело во входящем угле, не в методе.

## 15. Вклад структуры Рвачёва: структура ω^p·Φ против штрафа (`plate_solver/penalty.py`)
- КОРРЕКТНАЯ формулировка: «Ритц со структурой Рвачёва против Ритца со ШТРАФНЫМ
  учётом ГУ» (оба — один решатель Ритца, один базис Чебышёва; разница лишь в учёте
  ГУ). НЕ «метод без Рвачёва». Полигон — круг (шарнир), эталон — мягкий шарнир.
- Штраф: сырой базис ``w=Σcᵢφᵢ`` (БЕЗ ω) + ``γ∮_∂Ω w²ds`` к энергии каждой Пуассона.
- Числа (круг, Q=1024, эталон 3qa⁴/64D): целевая точность 1 % — структура берёт
  при ``N=9`` (p=2, cond=1.5e2), штраф при ``N=25`` (p=4, cond=5.8e5, γ=1e4). При
  ОДНОМ N=9 структура 0.08 % vs штраф 11 % (≈135×). Штраф также хуже обусловлен
  на каждом N (cond_штраф ≈ γ × cond_структ).
- **γ-«пол».** Малый γ (1e2) ⇒ ГУ не выполнены ⇒ штраф упирается в «пол» ~3.95 %
  (цель 1 % недостижима). Большой γ (≥1e6) ⇒ cond растёт без выигрыша в точности.
  Рабочий γ≈1e4. У структуры такого параметра НЕТ (ГУ тождественны) — в этом и
  вклад R-функций: точнее при меньшем N, лучше обусловлено, без настройки γ.

## 16. Штамп методом Ритца — смешанные ГУ и оператор отклика (`plate_solver/stamp_ritz.py`)
- ГУ функции Грина из `fix_base2.py` — НЕсимметричная пара: шарнир при x=0 (`w=0`,
  `w''=0`) + скользящая заделка при x=L (`w'=0`, `w'''=0`). В энергии `(D/2)∫(w'')²`
  СУЩЕСТВЕННЫ условия на `w` и `w'`: `w(0)=0` и `w'(L)=0`; `w''(0)=0` и `w'''(L)=0`
  выходят ЕСТЕСТВЕННО из минимума. Проверка вывода естественных ГУ — в докстринге.
- **Простой множитель `ω^p` НЕ годится:** `w(0)=0` (значение) и `w'(L)=0`
  (производная) — смешанные; `ω=x(L−x)` навязал бы лишнее `w(L)=0`. Берём БАЗИС из
  нуль-пространства двух линейных связей по Чебышёву: `Σc_i(−1)^i=0` (w(0)=0),
  `Σc_i i²=0` (w'(L)=0) — каждая функция базиса удовлетворяет обоим ГУ (вариант (б)).
- **Линейность ⇒ оператор отклика.** Изгиб Ритцем линеен ⇒ задаёт матрицу
  `M_ritz` (n+1×n+1): `w_узлы=M_ritz·(q−r)`, как и функция Грина. МОР поверх —
  ТОТ ЖЕ (нодальная реакция). Свободный прогиб (квартика `5qL⁴/24D`) — Ритц
  машинно (p≥4); Грин даёт его с ~0.78 % (трапеция). Контакт: Ритц↔Грин 0.27 %
  (p=16), Ритц↔Maple 2.28 % ≈ Грин↔Maple 2.44 % ⇒ Грин=Ритц=Maple.
- **Реакция «грубее» прогиба.** В малонагруженном хвосте контакта (x≳68) Ритц-реакция
  слегка ОСЦИЛЛИРУЕТ (Гиббс глобальных полиномов), пик у кромки и прогиб совпадают
  с Грином. Реакция — «шероховатая» величина для глобального Ритца (как момент в §14).

## 17. Круг как 1D-задача по радиусу (`plate_solver/radial.py`)
- Осесимметричная энергия защемления `πD∫[w''²+w'²/r²+2ν w''w'/r] r dr` (вывод из
  `(D/2)∫[(Δw)²−2(1−ν)det Hess]`; `Δw=w''+w'/r`). Структура `(a²−r²)²·Φ(r²)`:
  зануляет `w` и `w'` на `r=a`, ЧЁТНА по r ⇒ регулярна в центре (`w'(0)=0`).
- Члены `1/r, 1/r²` сингулярны, но при `w'(0)=0` интегранд×r РЕГУЛЯРЕН ⇒ Гаусс по
  ВНУТРЕННИМ узлам (r=0 не попадает) работает без спецприёмов.
- Шарнир — КАК В 2D, две радиальные Пуассоны (мягкий), структура `(a²−r²)·Φ(r²)`,
  форма Дирихле `∫u'v' r dr`. Точные решения (защемл. `(a²−r²)²·const`, мягкий
  `(a²−r²)(3a²−r²)·const`) ЛЕЖАТ в структурах ⇒ 1D машинно (p≥2). Числа: 1D↔точное
  0.000 %, 2D↔точное 0.10 % (квадратурный «пол» кривой границы), 1D↔2D 0.10 %.
  Вывод: 1D(полярн.)↔2D(RFM)↔аналитика сходятся; 1D даёт «точку отсчёта» без
  погрешности маски границы.

## 18. Точечная сила и уточнённая теория

- **Где δ законна.** Сосредоточенная сила P·δ(x−x₀) — непрерывный функционал
  на H² (энергия защемления, прямой Ритц по бигармонике: вложение H²⊂C⁰ в 2D),
  но НЕ на H¹: в расщеплении (P1) ``−ΔM = P·δ`` даёт ``M ~ (P/2π)·ln r`` —
  логарифмическая особенность, вне H¹-теории задачи Пуассона. Расщепление
  с истинной δ некорректно как вариационная задача первого кирпича.
- **Полиномы против ln r.** Глобальный чебышёвский базис приближает ln r
  (и w ~ r²ln r) лишь АЛГЕБРАИЧЕСКИ — спектральная скорость теряется.
  Численно (круг, точечная сила, Q=1024): rel(w_max) = 2.31 % (p=10) →
  1.44 % (p=16) → 1.11 % (p=22); при p>16 у расщепления cond(A) превышает
  машинную положительную определённость (Холецкий падает, §2). Поэтому
  ε-свип регуляризации немонотонен: базисная ошибка (~1 %) больше ошибки
  регуляризации при eps=0.025 (~0.2 %). Лечение — обогащение базиса
  функцией r²ln r (замечание к теореме 3; направление развития).
- **КТН под δ расходится.** КТН-прогиб содержит член ``c_curv·Δw``;
  под точечной силой ``Δw = −M/D ~ ln r`` ⇒ поправка логарифмически
  РАСХОДИТСЯ в точке приложения — как у Рейсснера–Миндлина: сдвиговая
  теория требует ПЯТНА нагрузки конечного размера. Поэтому point =
  регуляризованный patch (круговое пятно eps, q = P/(π·eps²)) — не
  компромисс реализации, а модельно-корректный выбор; дефолт
  eps = 0.05·min(ширина, высота bbox).
- **Зеркальная связь с контактом.** Тот же член ``c_curv·Δw`` в контактном
  условии СГЛАЖИВАЕТ реакцию: пик r_max на L-форме 112.6 (классика) →
  10.7 (КТН, табл. 4.3). Одна и та же физика поправки: уточнённая теория
  не терпит точечных концентраций ни в нагрузке, ни в реакции.

## 19. Канон напряжений на лицевых поверхностях

Источник: Ермоленко–Турков «Определение напряжений на лицевых поверхностях
пластины», формула (11); при b = 0 совпадает с
Михайловским–Бадокиным–Ермоленко (1999). Для линейного изгиба относительно
срединной плоскости (b = 0, мембранные T_ij ≡ 0) ядро формулы:

    σ_ii^{±h/2} = ± 6·M_ii / h²  +  ν/(1−ν) · q_n^{±},   i = 1, 2
    σ_12^{±h/2} = ± 6·M_12 / h²

(M_11 = Mx, M_22 = My, M_12 = Mxy = −D(1−ν)·w_xy). Член ν/(1−ν)·q_n^{±} —
вклад ПОПЕРЕЧНОГО ОБЖАТИЯ от нормального давления на соответствующей
лицевой поверхности. Редукция полного вида: члены с T входят как T_ii/h и
через смещение отсчёта момента b·T_ii — при T ≡ 0 исчезают при любом b
(sympy-тождество в tests/test_stresses.py; полный вид сверен с
первоисточником: мембранный член — T/h, изгибный и b-поправка — h̊²,
знак b-поправки противоположен изгибному).

Таблица знаков. Конвенция §0: q > 0 и w > 0 «вниз» ⇒ ось z сонаправлена
(вниз); ВЕРХНЯЯ лицевая (сторона внешней нагрузки) — z = −h/2, НИЖНЯЯ
(сторона основания/штампа) — z = +h/2:

| Величина | Верхняя лицевая (z = −h/2) | Нижняя лицевая (z = +h/2) |
|---|---|---|
| изгибная часть σ_ii | −6M_ii/h² | +6M_ii/h² |
| обжатие: q_n | внешняя нагрузка q (uniform/patch/point) | реакция r ≥ 0 в контактной зоне, иначе 0 |
| физический контроль в пролёте при q>0 | сжатие (σ < 0) | растяжение (σ > 0) |

Контроль знака: M = −D w'' > 0 в пролёте (w'' < 0 при прогибе «вниз») ⇒
σ_bot = +6M/h² > 0 — растяжение снизу ✓. Фиксируется 1D-тождеством B3-т1
(tests/test_stresses.py) — знаки НЕ изобретаются заново.

Предостережение (составные области): у R-операций (r_and/r_or) вторые
производные ω разрывны на линиях излома f₁ = f₂ (внутри области —
диагонали от углов); моменты из гессиана структуры в таких точках не
определены — на картах остаются NaN (маскируются; на 16×16-сетке L-формы
это единичные точки).

Согласование обозначений: Mx, My — из bending_moments (ladder.py, §14),
Mxy — bending_moments_full; у расщепления M-поле (P1) — след Mx+My
(§1), для напряжений используются ПОКОМПОНЕНТНЫЕ моменты из гессиана
структуры. Обжатие согласовано с KTNParams (ktn.py): те же h, ν.


## 20. Смешанные краевые условия: полная билинейная форма

При смешанных КУ энергия ОБЯЗАНА содержать гауссов член — полная форма

    a(w, v) = D ∬ [ Δw·Δv − (1−ν)·(w_xx·v_yy + w_yy·v_xx − 2·w_xy·v_xy) ] dΩ.

На защемлённой части границы (w = ∂w/∂n = 0) гауссов член — нуль-лагранжиан
(контурная дивергенция; потому clamped-путь v0.2 с ∫ΔwΔv был корректен);
на опёртой части он даёт естественное условие ИСТИННОГО шарнира M_n = 0.

Дискретная оговорка (важно): нуль-лагранжиан точен ТОЛЬКО при точной
квадратуре. Ступенчатая маска кривой границы и линии излома вторых
производных R-функций нарушают его на уровне квадратурного пола:
Δw(решение полной формы) отличается от ∫ΔwΔv-пути на rel ~3e-10 (rect,
полином-совместимая маска), ~2e-6 (круг), ~6e-4 (L-форма). Поэтому
унификация НЕ форсируется: защемление остаётся на ∫ΔwΔv (аналитически
корректном), полная форма — для смешанных КУ, где ей нет альтернативы.

Структура mixed-прямоугольника: w = (∏ ω_{c,i}²)·(∏ ω_{h,j})·Φ с
ПОЛИНОМИАЛЬНЫМИ множителями сторон (x−x₁), (x₂−x), (y−y₁), (y₂−y):
кинематика точна тождественно, домен = bbox ⇒ квадратура Гаусса точна,
изломов ω нет — форма работает с машинной точностью (Навье 7e-9,
Леви 7e-9 при p=12). Максимум симметричных постановок — в центре;
эталоны сравниваются В ЦЕНТРЕ (сравнение max-по-узлам против
max-по-сетке даёт ложные ~1e-4 расположения максимума).



## 21. Прогиб лицевых поверхностей и контактное условие

РЕШЕНИЕ (канон пакета): контактная кинематика — уточнённая теория КТН
(п. 21.1, формулы кода; первоисточник — 1D-теория школы); классическое
3D-восстановление (п. 21.2) — НЕЗАВИСИМАЯ ДИАГНОСТИКА, подтверждающая
поверхность и знаки. Формулы кода не меняются.

### 21.1. Канон: кинематика КТН (формулы кода)

Прогиб КОНТАКТИРУЮЩЕЙ (нижней) лицевой поверхности — этой величиной
проверяется зазор в контактном условии (ktn.contact_displacement,
синоним w_face_bottom):

    u_c = w + (2h_*² − h_Ψ²)·Δw − [h/(8(λ+2μ)) − h_*²/(μh) + h_*²h_z²/D]·q⁺
                                − [3h/(8(λ+2μ)) + h_*²/(μh) − h_*²h_z²/D]·D·r,

КТН-прогиб для w_max (ktn.corrected_deflection):

    w_KTN = w + (2h_*² − h_Ψ²)·Δw + [(3ν−1)h/(8(1−ν)E) − h_*²h_z²/D]·q⁺
                                  + [(3−ν)h/(8(1−ν)E) + h_*²h_z²/D]·D·r.

Согласование обозначений — §16/§18 (h_Ψ², h_*², h_z², c_curv из ktn.py);
редукция 2D → 1D при ∂/∂y = 0 совпадает с 1D-скриптом первоисточника
дословно (тождество редукции, tests/test_face_deflection.py). Ворота
«ручной расчёт по 21.1 ≡ путь решателя» (1e-12) — тест т7 там же.

Выходные ПОЛЯ лицевых поверхностей (fields.npz, схема 2) при
theory = ktn считаются ПО КАНОНУ 21.1: w_bot = u_c; канон выделяет
формулой только контактирующую лицевую, поэтому w_top ≡ w (срединная —
без смешения канонов), а dh = w_bot − w = каноническое смещение
контактирующей лицевой (в зоне контакта строго dh < 0 — обжатие/подъём
нижней поверхности к основанию; вне зоны знак определяется кривизным
членом). theory = classic: w_top ≡ w ≡ w_bot, dh ≡ 0.

### 21.2. 3D-восстановление (независимая диагностика)

Классическое прямое интегрирование по толщине: профиль σ_z — из
уравнения равновесия ∂σ_z/∂z = −div τ с параболическим τ и лицевыми
условиями σ_z(−h/2) = −q⁺, σ_z(+h/2) = −q⁻ (знаки §19); обжатие
ε_z = [σ_z − ν(σ_x + σ_y)]/E с изгибной частью σ_x + σ_y = 12(M_x+M_y)z/h³.
Интегрирование ε_z от срединной до лицевых даёт (sympy-вывод —
tests/test_face_deflection.py, т1–т5):

    w_bot^rec = w_mid + h_*²·Δw − h·(3 q⁺ + 13 q⁻) / (32 E),
    w_top^rec = w_mid + h_*²·Δw + h·(13 q⁺ + 3 q⁻) / (32 E),
    dh^rec    = −h·(q⁺ + q⁻) / (2 E) ≤ 0.

**Сверка с каноном.** Кривизный блок ТОЖДЕСТВЕНЕН:
2h_*² − h_Ψ² ≡ h_*² − h_z² — обжимной сдвиг лицевой (+h_*²·Δw)
плюс сдвиговая поправка срединного прогиба (−h_z²·Δw); тем самым
восстановление подтверждает ПОВЕРХНОСТЬ (нижняя лицевая) и ЗНАКИ канона.
q,r-члены различаются; точные символьные разности заморожены
постоянными воротами т6 (не ослаблять):

    Δb_q = b_q^rec − b_q^КТН = h(6ν³ + 14ν² + ν − 1) / (32E(ν − 1)),
    Δb_r = b_r^rec − b_r^КТН·D
         = h(Eh³ν² − 4Eh³ν + 2Eh³ − 26ν² + 52ν − 26) / (64E(ν − 1)²).

**Природа расхождения.** Восстановление 21.2 — ПОТОЧЕЧНОЕ (локальное)
восстановление обжатия по классическим профилям напряжений; кинематика
КТН — ВАРИАЦИОННОЕ определение параметров обжатия и сдвига (стеснённое
обжатие 1/(λ+2μ), сдвиговая податливость 1/(μh)). Оба описания —
поправки порядка O(h²) к классике, и их различие лежит в пределах
O(h²)-точности самой теории (числовая справка на замороженном снимке
L-контакта — cases/baselines.json, «lshape_ktn_recovery_vs_canon»:
разность порядка самой КТН-поправки). Для контакта и выходных полей
канон — 21.1.

## 22. Геометрическая нелинейность: итерация Кармана (`plate_solver/membrane.py`, v0.4.0)

**Дискретизация.** Прогиб `w = ω^m·Φ` (m = 2 защемление, m = 1 шарнир) и
перемещения в плане `u, v = s·Φ` (s = ω для immovable, s = 1 для movable).
Оператор изгиба `S_bend` (полная билинейная форма §20) и мембранная жёсткость
плоской задачи `K_uv` от прогиба НЕ зависят — факторизуются один раз. На каждом
шаге Пикара пересобирается лишь малая (N = (p+1)²) геометрическая жёсткость.

**Геометрическая жёсткость — симметричная форма.** Член `N:∇∇w` берётся
интегрированием по частям (при v = 0 на ∂Ω):

    K_geo[i,j] = ∫ N_αβ ψ_{i,α} ψ_{j,β} dΩ
               = ∫ (N_x ψ_{i,x}ψ_{j,x} + N_xy(ψ_{i,x}ψ_{j,y}+ψ_{i,y}ψ_{j,x})
                    + N_y ψ_{i,y}ψ_{j,y}) dΩ.

Симметричная форма НЕ требует дискретного равновесия `N` (в отличие от
`∫N:(∇∇ψ_i)ψ_j`) и положительно определена при растяжении — мембранное
ужесточение. Мембранная подзадача: `K_uv·[a;b] = f_e` с предварительными
деформациями `e = (½w_x², ½w_y², w_x w_y)`; immovable ⇒ K_uv положительно
определена (Холецкий), movable ⇒ ядро жёстких мод, решаем МНК (правая часть им
ортогональна).

**Кандидат в теорему T5 (сходимость нелинейной итерации).** Отображение Пикара
`w_{k+1} = T(w_k)` на шаре `{‖w‖ ≤ R}` энергетического пространства — сжатие при
нагрузке ниже порога: нелинейный член квадратичен по `w`, `T` липшицев с
константой `L(R, q)`; при малой `q` (или малом шаге нагрузки) `L < 1`. Отсюда
локальные существование, единственность и геометрическая сходимость; шаги по
нагрузке (`n_load_steps`) продолжают решение по параметру до `w/h ~ 1–2` и делают
достижимым мембранный предел (Gate M, `w/h ~ 6`). С T2 (сходимость Ритца
подзадач) — сходимость дискретизованного нелинейного решения. Ускорение Андерсона
(смешивание истории невязок) сохраняет ту же неподвижную точку, но заменяет
линейную сходимость сверхлинейной. Это расширяет аппарат в нелинейный режим.

**Аргумент валидации.** Совпадение решателя с эталонами РАЗНОЙ математической
природы (точное аналитическое Hencky; ряды Way; энергия Ритца Тимошенко; ряды
Фурье Levy; МКЭ/IGA) исключает циклическую самопроверку и служит сильным
аргументом достоверности (числа — `benchmarks.py`, роли — THEORY.md разд. 7).

**Приложение B — нормировки (риск №1: разные ν у эталонов).** Безразмерная
нагрузка `P̄ = p a⁴/(E h⁴)` (круг: a — радиус; квадрат: a — сторона); прогиб —
`w/h`; `D = E h³/[12(1−ν²)]`. Пересчёт: `p a⁴/(D h) = 12(1−ν²)·P̄`,
`p a⁴/(64 D h) = [12(1−ν²)/64]·P̄` (ν=0.3 → 0.1706·P̄). ν эталонов различаются
(0.30 у Way/Тимошенко/Hencky, 0.316 у Levy) → систематический сдвиг ~2–4 %;
каждый тест ЯВНО задаёт свой ν, эталоны с разными ν не смешиваются в одном
допуске. Все пересчёты — через `benchmarks.py`.

## 23. Полная нелинейная КТН: редукционная верификация (теорема T6, v0.5.0)

**Кандидат в теорему T6 (структурная согласованность / редукция).**
Дискретизованная задача КТН (`ktn_full.KTNPlate`) НЕПРЕРЫВНО зависит от
параметров толщины `h_ψ², h_*²` и в пределе `(h_ψ², h_*²) → 0` (со снятием
поля `ψ`) редуцируется к дискретной задаче Кармана, а далее — к Кирхгофу;
КТН-поправка есть `O(h²/L²)` и исчезает при `h/L → 0`. Вместе с T5 (сходимость
нелинейной итерации, переносится с Кармана на КТН) это верифицирует решатель ПО
ПОСТРОЕНИЮ против всех подтеорий.

Редукционная лестница (`tests/test_ktn_full.py`), сильнее одного эталона —
демонстрирует корректную редукцию:

* **R1** — при выключенных КТН-членах `ktn_full ≡ karman` до машинной точности
  (прямая проверка, что члены (A), (B) собраны без ошибок знака/множителя);
  реализовано флагом `include_ktn_terms=False` ⇒ тот же код-путь Кармана.
* **R2** — тонкая пластина + малый прогиб ⟹ `ktn_full → classic` (Кирхгоф).
* **R4** — гашение поправки: при ФИКСИРОВАННОЙ `P̄` эффект КТН/Карман ∝ `h²`
  (эмпирически `effect/h² = const`), при `h/L = 0.02` эффект < 1 %.
* **R5** — лицевые `ktn_full` при малом прогибе ≈ `ktn_linear` (смыкание старой
  и новой реализаций).

Совпадение с подтеориями РАЗНОЙ природы (аналитический Кирхгоф; кармановские
литературные эталоны Hencky/Way/Levy; линейные лицевые поправки `ktn_linear`)
исключает циклическую самопроверку — сильный довод достоверности.

**Gate R3 (1D-КТН ↔ 2D, §9) — ОТЛОЖЕН.** Продолжение несократимого ядра
1D↔2D на КТН требует проверенной ОДНОМЕРНОЙ КТН-кривой школы (первоисточники —
недоступны в этом окружении); по порядку сокращения (§14) допустимо свести к
одной контрольной точке позже, когда 1D-КТН эталон будет под рукой. Линейный
1D↔2D (`classic`, сакральные ворота) остаётся зелёным и не ослабляется.

**Зависимость эффекта от толщины.** Член обжатия `h_*²Δw` — подпись КТН в
лицевых напряжениях (ненулев даже под равномерной нагрузкой, т.к. `Δw = −M/D`
меняется по области); член `−h_*²Δq` — подпись в срединном прогибе под
НЕравномерной нагрузкой (растёт с локализацией). Интроспекция `h_ψ², h_*², h_c²`
и `h/L` — `Result.thickness_params` (§6.3).

## 24. Нелинейный контакт МОР+КТН: теорема T7 (v0.6.0)

Уникальная связка «полная нелинейная КТН + контакт по МОР» на произвольной, в
т.ч. многосвязной, области — кульминация научного ядра (в литературе связки нет).
Контакт «щупает» ЛИЦЕВУЮ поверхность (условие Синьорини на грани, §4.1):
`u_c = w + (h_c²−h_*²)Δw ≤ z`. Коэффициент кривизны масштабируется теорией
(`TheoryParams.face_curv_coeff`): 0 для classic/karman (контакт по срединной
`u_c = w`), физический для полной КТН (подпись КТН в контакте — сглаживание
реакции сдвигом/обжатием).

**Две схемы композиции** (`contact_nl.py`):

* **вложенная** (эталон корректности): внешний цикл МОР, внутри — полный
  сходящийся `KTNSolver` на каждой итерации (тёплый старт предыдущим прогибом);
  проста, надёжна, дорога;
* **совмещённая** (рабочая): ОДИН шаг Пикара на ОДИН шаг МОР с общей
  недорелаксацией — в разы быстрее (переиспользует `_picard_map`).

**Кандидат в теорему T7 (сходимость совмещённого МОР–КТН на произвольной, в т.ч.
многосвязной, области).** Совмещённое отображение — КОМПОЗИЦИЯ сжатия МОР (T4,
Михайловский–Тарасов: `0 < β < 2`) и сжатия Пикара (T5, нелинейная итерация); при
условии малости (нагрузка/зазор) композиция двух сжатий — сжатие, откуда
сходимость и единственность. На многосвязной области полнота системы `{ω^s φ}`
(T1) и сходимость Ритца (T2) сохраняются для R-конъюнкции `ω` (R-операции над ω,
§5.1), откуда сходимость дискретизации. Это прямое ОБОБЩЕНИЕ T4 на нелинейный
контакт — сильнейший теоретический результат работы. Вложенный решатель —
КОНСТРУКТИВНЫЙ эталон; совмещённый — предмет теоремы. Гейт R4 (совмещённый =
вложенному до допуска) — эмпирическое подтверждение.

**Усиление T6 единой моделью.** Все четыре теории — пресеты параметров ОДНОГО
решателя (`theory.py`, `ktn_solver.py`): редукции ТОЧНЫ ПО ПОСТРОЕНИЮ (тот же
путь исполнения), а не «совпадением до машинной точности» — структурная
согласованность теорий становится точным утверждением. Редукции контакта: контакт
выкл (`r≡0`) → решатель КТН v0.5 (R1); уточнение выкл (`refinement_scale=0`) →
кармановский контакт (R3); всё — по единой модели.
