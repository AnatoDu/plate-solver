# Руководство по миграции

## v0.5.0 — переименование теорий (`ktn` → `ktn_linear` / `ktn_full`)

С введением ПОЛНОЙ нелинейной теории Кармана–Тимошенко–Нагди значение
`[model] theory = "ktn"` стало неоднозначным: оно означало ЛИНЕЙНЫЕ поправки
сдвига/обжатия постобработкой, а «КТН» — имя полной (теперь нелинейной) теории.
Введены ЯВНЫЕ имена без тихой смены семантики.

| Старое | Новое | Что это |
|---|---|---|
| `theory = "ktn"` | `theory = "ktn_linear"` | линейные поправки сдвига/обжатия (прежнее поведение, **число-в-число**) |
| — (не было) | `theory = "ktn_full"` | полная нелинейная КТН (Карман + сдвиг + обжатие) |

**Обратная совместимость.** `theory = "ktn"` работает как **депрекация-алиас**
на `ktn_linear` (старые case-файлы считают то же, байт-в-байт) и выводит
`DeprecationWarning`. Алиас `ktn` НЕ переведён на `ktn_full` — это тихо сменило
бы результат прежних расчётов. **Алиас будет удалён в v1.0.0.**

### Как обновить case-файл

Было:

```toml
[model]
theory = "ktn"           # неоднозначно
h = 0.06
```

Стало (эквивалентно, без предупреждения):

```toml
[model]
theory = "ktn_linear"    # явно: линейные поправки
h = 0.06
```

Чтобы перейти на ПОЛНУЮ нелинейную теорию (только защемление, канонические
области):

```toml
[model]
theory = "ktn_full"
inplane_bc = "immovable" # закрепление кромки в плане
h = 0.06
n_load_steps = 4         # большой прогиб — шаги по нагрузке
```

### CLI

Флаги `--theory` / `--inplane-bc` принимают новые имена:
`--theory ktn_linear` | `--theory ktn_full`. Устаревшее `--theory ktn` из CLI
исключено (используйте явные имена); алиас `ktn` сохранён только для
case-файлов ради обратной совместимости.

### Программный API

`ktn.KTNParams` (линейные коэффициенты) и `ktn.stresses_faces` — без изменений
(контактный тракт). Новый первоклассный слой лицевых величин — `faces.FaceParams`
(канонические имена параметров толщины `h_psi_sq`, `h_star_sq`, `h_c_sq`).
Полный нелинейный решатель — `ktn_full.KTNPlate`.
